Highlight #1 What is VCS?

VCS stands for Version Control System and is a system that allows users to keep track of various iterations their various files. This allows users to maintain better control over the different versions of their files for their project.

Highlight #2 GitHub: The Basics

There are many different pieces to GitHub with one of the main aspects being local and remote repositories. These repositories serve as the warehouses for the various types of files relating to a project. After making changes to a local version of the file, the new updated file can be "pushed" up to the remote repository for file backup, version tracking, and collobration with other members of the project.

Highlight #3 Publishing your files to Git

One of the key components of a VCS such as Github is publishing your files to the system. The first part of this process is making sure your remote repository is setup in GitHub. Second, the remote repository needs to be added in the local Git system so Git knows where to send the files. Third, the "git push" commands needs to be run to tell Git to send the files to the remote repository.
